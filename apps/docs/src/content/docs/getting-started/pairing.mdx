## What is Pairing?

Pairing is the process of establishing a trusted relationship between two Warren nodes. After pairing:

- The host assigns the controller a unique **Device ID**
- Both sides store a **shared secret** (derived via X25519 ECDH in v0.2+)
- Future connections authenticate automatically — no QR code needed

Pairing is **explicit and one-time**. You pair each device to each host individually. There is no "pairing with Warren" — there is only pairing device A to device B.

## Pairing Flow (v0.2+)

```
Controller                        Host
    │                               │
    │  1. Scan QR (IP + nonce)      │
    │──────────────────────────────>│
    │                               │
    │  2. Send: pub_key + nonce_sig │
    │──────────────────────────────>│
    │                               │
    │  3. Host verifies nonce       │
    │     Performs ECDH             │
    │     Stores shared_secret      │
    │     Assigns device_id         │
    │                               │
    │  4. Send: pub_key + device_id │
    │<──────────────────────────────│
    │                               │
    │  5. Controller derives        │
    │     shared_secret             │
    │     Stores: host + secret     │
    │                               │
    │  6. Connection established    │
    │<═══════════════════════════>  │
```

<Aside type="note">
In v0.1, pairing uses a static token instead of X25519. Full cryptographic pairing arrives in v0.2.
</Aside>

## Managing Paired Devices

### View paired devices

Open the Warren dashboard (tray icon → Open Dashboard) → **Devices** tab.

Each device shows:
- Name (machine hostname or user-assigned name)
- Status (online / last seen timestamp)
- Permission level (full / revoked)

### Revoke a device

Click **Revoke** next to the device. This:
1. Deletes the shared secret from the host
2. Closes any active sessions from that device
3. Future connection attempts from that device will fail

The device can be re-paired at any time by showing a new QR code.

### Pair a new device

Tray icon → **Show QR Code**, or in the Dashboard → Devices → **Pair New Device**.

## Security Properties

| Property | Detail |
|----------|--------|
| Per-pair secrets | Each device pair has its own independent secret |
| No transitivity | Access to Mac A does not grant access to Mac B |
| Forward secrecy | TODO(v0.2): Ratchet mechanism for future sessions |
| Revocation | Instant — no grace period, no sync required |

See [Security Model](/security/model/) for full details.
